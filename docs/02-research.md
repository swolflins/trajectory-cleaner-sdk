## 1. 调研概述

### 1.1 调研目的

为轨迹清洗 SDK 的设计与实现提供技术选型依据，明确：

1. 轨迹数据存在的质量问题及其成因
2. 业界主流的四大处理能力（去噪、抽稀、绑路、补偿）及其算法
3. 大厂云端方案的能力边界与参数设计
4. 开源绑路方案的能力与适用性
5. 学术界经典算法的可借鉴点
6. 本项目应实现的范围与简化策略

### 1.2 调研范围

| 维度 | 覆盖 |
|------|------|
| 能力维度 | 去噪、抽稀、绑路、补偿（全链路） |
| 厂商维度 | 腾讯轨迹云、高德猎鹰、百度鹰眼、阿里云、华为定位 |
| 开源维度 | OSRM、GraphHopper、Valhalla |
| 学术维度 | Douglas-Peucker、Visvalingam-Whyatt、HMM 地图匹配、卡尔曼滤波 |
| 实现维度 | 仅去噪 + 抽稀（绑路 / 补偿因依赖路网不实现） |

### 1.3 调研方法

1. 官方文档与 API 参考阅读
2. 开源项目源码与文档研读
3. 学术论文检索（Google Scholar / ACM / IEEE）
4. 对比维度：能力覆盖、参数设计、精度声称、性能、依赖、成本

## 2. 轨迹数据特征与问题分类

### 2.1 GNSS 定位原理简述

GNSS（Global Navigation Satellite System）通过接收多颗卫星信号计算接收机位置。定位误差来源：

| 误差源 | 量级 | 特征 |
|--------|------|------|
| 电离层延迟 | 5-15 米 | 可建模校正 |
| 对流层延迟 | 2-5 米 | 可建模校正 |
| 多径效应（Multipath） | 0-50 米 | 城市峡谷严重，难校正 |
| 卫星几何分布（DOP） | 1-10 倍系数 | HDOP 影响水平精度 |
| 接收机噪声 | 0.5-2 米 | 高端设备可忽略 |
| 钟差 | 已通过差分校正 | RTK 可降至厘米级 |

消费级 GPS 设备在城市环境典型精度 5-30 米，开阔环境 3-10 米，遮挡环境可达 50 米以上甚至失锁。

### 2.2 五大质量问题

#### 2.2.1 漂移（Drift）

**定义**：定位点缓慢偏离真实位置，通常表现为连续多个点向同一方向偏移。

**成因**：
- 多径效应：信号经建筑物反射后到达接收机
- 卫星几何分布差（HDOP 高）
- 弱信号下的卡尔曼滤波发散

**特征**：
- 单点位移小，但累积偏离大
- 速度与方向变化平缓，难以用速度约束检测
- 在城市峡谷、隧道出入口、室内高发

**对下游影响**：
- 里程虚增（绕路假象）
- 围栏误触发（设备实际在内但定位在外）
- 抽稀后形态失真

**处理思路**：
- 绑路可修正（投影到路网）
- 卡尔曼滤波可平滑
- 纯去噪难以根治

#### 2.2.2 飞点（Outlier / Spike）

**定义**：速度或加速度异常的孤立定位点，表现为瞬时跳变。

**成因**：
- 卫星切换导致瞬间坐标重算
- 接收机冷启动 / 热启动
- 信号短暂丢失后重新捕获
- 设备时钟跳变

**特征**：
- 单点偏离大，相邻点位移可达数百米
- 推算速度远超物理可能（如 200 m/s）
- 加速度异常（如 50 m/s²）
- 通常为孤立单点，少数为连续 2-3 点

**对下游影响**：
- 里程暴增（飞点贡献大量虚假距离）
- 速度统计失真（最大速度异常）
- 轨迹形态破坏（出现尖刺）

**处理思路**：
- 速度约束（物理上限）
- 统计方法（Z-score / IQR）
- 卡尔曼滤波新息（innovation）检测

#### 2.2.3 静止抖动（Static Jitter）

**定义**：设备实际停止但 GPS 持续输出小幅噪声点。

**成因**：
- GPS 即使静止也会持续输出（受卫星运动与噪声影响）
- 设备未做静止判定
- 上传策略为定时上传而非位移触发

**特征**：
- 点位在真实位置周围随机抖动，半径 5-30 米
- 速度不为 0（因抖动产生虚假速度 0.5-3 m/s）
- 持续时间长（停车 1 小时可能产生数百点）
- 抖动点时间间隔规律

**对下游影响**：
- 静止段里程虚增（抖动点累加距离）
- 停留点识别困难（抖动产生虚假移动）
- 数据膨胀（静止段点数占比可达 50%+）

**处理思路**：
- 位移阈值 + 时间窗口判定静止
- 状态机平滑（本项目核心能力）
- 静止段仅保留锚点

#### 2.2.4 信号中断（Signal Loss）

**定义**：遮挡环境下定位丢失，导致轨迹时间断档。

**成因**：
- 隧道、地下停车场、室内
- 城市峡谷卫星被遮挡
- 设备省电模式间歇关 GPS

**特征**：
- 相邻点时间间隔远大于正常采样间隔
- 断档段无数据
- 恢复后首点可能漂移

**对下游影响**：
- 轨迹不连续
- 里程低估（断档段未计算）
- 速度失真（断档后首点速度基于长时间差计算，偏低）

**处理思路**：
- 补偿（插值或路径规划）
- 绑路可借助路网还原

#### 2.2.5 冗余点（Redundant Points）

**定义**：直线段或低速段内大量重复的近距点。

**成因**：
- 定时上传（如每 5 秒一上传）而非位移触发
- 直线高速行驶时点密度高
- 低速 / 静止时点密度高

**特征**：
- 相邻点距离 < 抽稀阈值
- 信息冗余（位置变化小）
- 点数膨胀，存储与传输成本高

**对下游影响**：
- 数据冗余，存储成本高
- 回放卡顿
- 下游计算（如绑路）计算量大

**处理思路**：
- 抽稀算法（Douglas-Peucker、Visvalingam-Whyatt、距离阈值）

### 2.3 问题分类矩阵

| 问题 | 检测方法 | 处理能力归属 | 本项目是否处理 |
|------|----------|-------------|---------------|
| 漂移 | 难检测，需绑路 | 绑路 / 滤波 | 部分（异常检测可剔除极端漂移） |
| 飞点 | 速度/加速度约束、统计 | 去噪 | 是 |
| 静止抖动 | 位移阈值 + 状态机 | 去噪（伪静止） | 是 |
| 信号中断 | 时间间隔检测 | 补偿 | 否（不实现补偿） |
| 冗余点 | 距离阈值、形态算法 | 抽稀 | 部分（静止段合并） |

## 3. 能力一：去噪（Denoising）

去噪是轨迹清洗的核心能力，目标是识别并处理飞点与静止抖动。本项目去噪分为三个子能力：精度过滤、伪静止剔除、异常检测。本节聚焦异常检测的算法选型。

### 3.1 速度约束法（Velocity Constraint）

**原理**：基于物理上限判定，若推算速度超过合理阈值则判为飞点。

**判定规则**：
- 给定相邻点 P1(t1, x1, y1)、P2(t2, x2, y2)
- 计算位移 d = haversine(P1, P2)
- 计算时间差 Δt = t2 - t1
- 推算速度 v = d / Δt
- 若 v > MaxSpeed（如 50 m/s = 180 km/h）则 P2 判为飞点

**优点**：
- 计算简单，无需统计建模
- 物理意义明确
- 适用于冷启动（无历史数据）

**缺点**：
- 阈值难定：阈值过松漏检，过严误杀（如航空、高铁场景）
- 无法检测"小飞点"（速度未超阈值但仍是异常）
- 依赖时间戳准确性

**本项目应用**：作为 Stage 3 的前置硬约束，超过 `MaxSpeed` 直接判飞点，不进入统计判定。

### 3.2 Z-score 方法

**原理**：基于正态分布假设，若数据点偏离均值超过 k 倍标准差则视为异常。

**公式**：

$$Z = \frac{|v - \mu|}{\sigma}$$

其中 μ 为速度均值，σ 为速度标准差。当 Z > 阈值（默认 2.0，对应 95% 置信区间外）判为飞点。

**冷启动校准**：
1. 取轨迹前 N 个点（N = CalibrationSize，默认 50）
2. 计算这些点的速度序列
3. 用 3σ 原则先剔除明显飞点，得到干净基线
4. 计算基线 μ 与 σ

**持续检测**：
- 滑动窗口（大小 100）维护近期速度序列
- 每来一个新点，更新窗口并重算 μ、σ
- Z-score 判定

**优点**：
- 计算高效（O(1) 增量更新）
- 适应轨迹速度变化（滑动窗口）
- 阈值有统计意义

**缺点**：
- 假设速度服从正态分布，实际轨迹速度分布常偏态或多峰
- 对极端飞点敏感（单个飞点拉高 σ，降低后续检测灵敏度）
- 需要冷启动样本

**级联误报防护**：
- 问题：连续飞点会污染滑动窗口的 μ、σ，导致后续正常点被误判
- 方案：维护 `lastActualPoint`，飞点命中后进入 grace period，期间不复位基线

### 3.3 IQR 方法（Interquartile Range）

**原理**：基于四分位数的非参数方法，不假设分布形态。

**计算**：
1. 对速度序列排序
2. 计算 Q1（25% 分位数）、Q3（75% 分位数）
3. IQR = Q3 - Q1
4. 上界 = Q3 + k·IQR（默认 k=1.5）
5. 下界 = Q1 - k·IQR
6. 速度超出上下界判为飞点

**优点**：
- 不假设分布形态，对偏态分布稳健
- 对极端值不敏感（基于分位数而非均值）
- 阈值系数 k 有经验值（1.5 为温和，3.0 为激进）

**缺点**：
- 排序计算开销大（O(n log n)），滑动窗口下需重排
- 小样本下分位数估计不稳定
- 不如 Z-score 直观

**本项目应用**：作为 Z-score 的备选方法，通过 `DetectorMethod` 配置切换。

### 3.4 卡尔曼滤波（Kalman Filter）

**原理**：基于状态空间模型，融合预测与观测，递归估计真实位置。

**状态向量**：`[x, y, vx, vy]`（位置 + 速度）

**预测方程**：

$$\hat{x}_{k|k-1} = F \cdot \hat{x}_{k-1|k-1}$$

$$P_{k|k-1} = F \cdot P_{k-1|k-1} \cdot F^T + Q$$

**更新方程**：

$$K_k = P_{k|k-1} \cdot H^T \cdot (H \cdot P_{k|k-1} \cdot H^T + R)^{-1}$$

$$\hat{x}_{k|k} = \hat{x}_{k|k-1} + K_k \cdot (z_k - H \cdot \hat{x}_{k|k-1})$$

$$P_{k|k} = (I - K_k \cdot H) \cdot P_{k|k-1}$$

其中 F 为状态转移矩阵，H 为观测矩阵，Q 为过程噪声协方差，R 为观测噪声协方差。

**飞点检测**：
- 新息（innovation）`ν = z_k - H · x̂_{k|k-1}`
- 若 `ν^T · S^{-1} · ν > χ²` 阈值（S 为新息协方差），判为飞点
- 飞点时不更新状态（拒绝观测）

**优点**：
- 同时平滑轨迹与检测飞点
- 可融合速度、航向等多源信息
- 适合实时流式处理

**缺点**：
- 需调参（Q、R 矩阵），调参困难
- 假设线性高斯模型，强非线性场景失效
- 实现复杂度高
- 状态发散问题（需 Reset 机制）

**本项目取舍**：不采用卡尔曼滤波。原因：
1. 实现复杂，双语言对等难度大
2. 调参依赖场景，难以提供通用默认值
3. 卡尔曼滤波更适合"平滑"而非"清洗"，平滑会改变所有点位置，不符合"保留有效原始点"的目标

### 3.5 其他去噪方法

#### 3.5.1 中值滤波（Median Filter）

对滑动窗口内坐标取中值，可有效去除孤立飞点。但会引入延迟，且改变所有点位置。适合后处理平滑，不适合清洗。

#### 3.5.2 滑动平均（Moving Average）

对窗口内坐标取均值，平滑效果好但会过度平滑，丢失细节。同样改变所有点位置。

#### 3.5.3 基于密度的异常检测（DBSCAN / LOF）

将轨迹点视为空间点云，用密度方法识别离群点。适合静止抖动检测，但对运动中的飞点效果差，且计算开销大。

### 3.6 去噪方法选型总结

| 方法 | 复杂度 | 是否改变正常点 | 适合场景 | 本项目 |
|------|--------|---------------|----------|--------|
| 速度约束 | O(1) | 否 | 冷启动硬过滤 | 是（前置） |
| Z-score | O(1) 增量 | 否 | 通用 | 是（主方法） |
| IQR | O(n log n) | 否 | 偏态分布 | 是（备选） |
| 卡尔曼滤波 | O(1) | 是（平滑） | 实时平滑 | 否 |
| 中值/均值滤波 | O(n) | 是（平滑） | 后处理平滑 | 否 |
| DBSCAN / LOF | O(n²) | 否 | 静止抖动 | 否（开销大） |

## 4. 能力二：抽稀（Simplification / Compression）

抽稀目标是在保持轨迹形态前提下减少点数，降低存储与计算成本。

### 4.1 Douglas-Peucker 算法（DP 算法）

**原理**：基于垂直距离的递归分治算法。

**算法步骤**：
1. 连接轨迹首尾点 A、B 形成弦
2. 找到中间所有点到弦 AB 的最大垂直距离 d_max 及对应点 C
3. 若 d_max < 阈值 ε，则丢弃 A、B 之间所有点
4. 否则保留 C，对 AC 段与 CB 段递归

**伪代码**：
```
function DP(points, epsilon):
    if len(points) < 3:
        return points
    A, B = points[0], points[-1]
    dmax, idx = 0, 0
    for i in 1..len(points)-2:
        d = perpendicular_distance(points[i], A, B)
        if d > dmax:
            dmax, idx = d, i
    if dmax > epsilon:
        left = DP(points[0..idx], epsilon)
        right = DP(points[idx..end], epsilon)
        return left + right[1:]
    else:
        return [A, B]
```

**优点**：
- 保形效果好（保留拐点）
- 算法经典，实现成熟
- 时间复杂度 O(n log n) 平均，O(n²) 最坏

**缺点**：
- 不自顶向下考虑时间维度（仅几何）
- 对自相交轨迹效果差
- 阈值 ε 需人工调参
- 非增量式（需完整轨迹）

**改进变体**：
- Topological DP：考虑拓扑一致性
- Time-aware DP：加入时间约束
- streaming DP：流式版本

### 4.2 Visvalingam-Whyatt 算法（VW 算法）

**原理**：基于有效面积（Effective Area）的渐进式删除。

**算法步骤**：
1. 对每个中间点 P_i，计算其与前一点 P_{i-1}、后一点 P_{i+1} 形成三角形的面积
2. 将所有点按面积升序排序
3. 逐个删除面积最小的点，删除后更新相邻点面积
4. 直到所有点面积 > 阈值或达到目标点数

**优点**：
- 保留视觉显著的拐点（面积大者）
- 比 DP 更符合视觉感知
- 可指定目标点数（而非仅阈值）

**缺点**：
- 计算开销比 DP 大（需动态维护优先队列）
- 对短轨迹效果不明显

### 4.3 距离阈值法

**原理**：若相邻点距离小于阈值，则丢弃后一点。

**变体**：
- **简单距离阈值**：`if dist(P_i, P_{i-1}) < threshold: drop P_i`
- **时间-距离联合**：`if dist < d_thr and dt < t_thr: drop`
- **角度-距离联合**：直线段容忍更大间距，拐点处密集

**优点**：
- 计算极简 O(n)
- 流式友好，可单点增量
- 适合冗余点剔除

**缺点**：
- 保形效果差（可能删除重要拐点）
- 阈值难定

**本项目应用**：伪静止状态机的静止段合并本质上是距离阈值法的一种特化（位移小于 `MinDisplacement` 视为冗余）。

### 4.4 Reumann-Witk 算法

**原理**：以相邻两点建立基线，向两侧扩展带宽区域，落入带宽内的点丢弃。

**特点**：流式友好，O(n)，适合实时。但保形效果一般。

### 4.5 抽稀方法选型

| 方法 | 时间复杂度 | 流式 | 保形 | 本项目 |
|------|-----------|------|------|--------|
| Douglas-Peucker | O(n log n) | 否 | 好 | 调研参考，暂不实现独立 Stage |
| Visvalingam-Whyatt | O(n log n) | 否 | 优 | 调研参考，暂不实现 |
| 距离阈值 | O(n) | 是 | 一般 | 是（伪静止合并的副产物） |
| Reumann-Witk | O(n) | 是 | 一般 | 否 |

**说明**：本项目当前需求聚焦去噪，抽稀仅作为伪静止剔除的副产物实现（静止段合并）。完整 DP / VW 抽稀作为扩展能力预留，详见 [03-design.md](./03-design.md) 扩展性设计。

## 5. 能力三：绑路（Map Matching）

绑路是将 GPS 点序列投影到路网拓扑上，还原真实行驶路径。这是大厂能力的核心，依赖路网数据。

### 5.1 几何匹配法（Geometric Matching）

**原理**：对每个 GPS 点，找到路网中几何距离最近的道路段，投影到该段。

**方法**：
- 点-线最近距离计算
- 投影到线段

**优点**：实现简单，O(n · m)（n 轨迹点，m 路段）

**缺点**：
- 忽略路网拓扑（可能跳到平行路）
- 忽略连续性（相邻点可能匹配到不相邻路段）
- 拓扑错误率高

### 5.2 拓扑匹配法（Topological Matching）

**原理**：在几何匹配基础上，考虑路网连通性，确保匹配路段在拓扑上连续。

**方法**：
- 候选路段集合（几何近邻）
- 在路网图上做最短路径连接相邻匹配段
- 选择拓扑可行的匹配序列

**优点**：减少跳变，匹配更合理

**缺点**：仍可能选错候选路段

### 5.3 Hidden Markov Model（隐马尔可夫模型）

**原理**：将"真实路段"视为隐状态，GPS 点为观测，用 HMM 建模。

**模型定义**：
- 隐状态 S_i：第 i 个 GPS 点对应的真实路段
- 观测 O_i：第 i 个 GPS 点
- 发射概率 P(O_i | S_i)：GPS 点到路段的投影距离（高斯分布）
- 转移概率 P(S_{i+1} | S_i)：相邻匹配路段间的路网最短路径与直线距离的吻合度

**发射概率**：

$$P(z_t | r_i) = \frac{1}{\sqrt{2\pi}\sigma} \exp\left(-\frac{d(z_t, r_i)^2}{2\sigma^2}\right)$$

其中 d(z_t, r_i) 为 GPS 点到路段的投影距离，σ 为 GPS 精度。

**转移概率**：

$$P(r_j | r_i) = \frac{d_{route}(r_i, r_j)}{|d_{great}(z_t, z_{t+1})|}$$

其中 d_route 为路网最短路径距离，d_great 为大圆距离。比值越接近 1 越合理。

**Viterbi 算法**：求解最优隐状态序列。

$$\delta_t(j) = \max_i [\delta_{t-1}(i) \cdot P(r_j | r_i)] \cdot P(z_t | r_j)$$

**优点**：
- 综合考虑几何与拓扑
- 对 GPS 噪声鲁棒
- 学术与工业界主流方法（被 Google、Microsoft、OSRM 等采用）

**缺点**：
- 实现复杂
- 需路网数据
- 计算量大（候选路段组合爆炸）

**经典论文**：
- Newson & Krumm (2009), "Hidden Markov Map Matching Through Noise and Sparseness", Microsoft Research
- 该论文是工业界绑路的事实标准参考

### 5.4 多假设方法（Multiple Hypothesis Techniques）

**原理**：维护多个候选匹配路径，逐步淘汰不可能的假设。

**特点**：介于拓扑匹配与 HMM 之间，工程上易实现。

### 5.5 高级方法

- **粒子滤波（Particle Filter）**：用粒子集合表示位置分布，适合非线性非高斯
- **条件随机场（CRF）**：判别式模型，特征工程灵活
- **深度学习**：用序列模型（LSTM / Transformer）学习匹配，需大量标注数据

### 5.6 绑路能力选型

本项目**不实现绑路**。原因：
1. 强依赖路网数据（OSM 或大厂路网）
2. 路网数据处理、空间索引（R-tree / GeoHash）、最短路径计算（Dijkstra / A*）工程量大
3. HMM + Viterbi 实现复杂
4. 这些能力属于大厂核心壁垒，开源方案（OSRM / GraphHopper / Valhalla）已提供成熟实现

建议上层应用接入开源绑路引擎（见第 8 节）。

## 6. 能力四：补偿（Supplement / Interpolation）

补偿在信号中断段插入合理点，使轨迹连续。

### 6.1 线性插值

**原理**：在断档两端点间线性插值。

**方法**：
- P1(t1, x1, y1)、P2(t2, x2, y2) 间断档
- 按采样间隔 Δt 生成中间点
- 经纬度按时间线性插值

**优点**：实现简单

**缺点**：
- 直线插值不考虑道路，可能穿建筑
- 不考虑运动学（速度、加速度）
- 适合短断档（< 30 秒）

### 6.2 路径规划插值

**原理**：在断档两端点间做路网最短路径规划，沿路径插值。

**方法**：
- 将 P1、P2 绑路到路网
- 用 Dijkstra / A* 计算最短路径
- 沿路径按采样间隔插值

**优点**：符合实际行驶路径

**缺点**：
- 依赖路网数据与绑路能力
- 计算开销大
- 适合长断档（> 30 秒）

### 6.3 航位推算（Dead Reckoning）

**原理**：利用上一已知点的速度、航向，结合惯性传感器（IMU）推算位置。

**方法**：
- 位置递推：`x_{t+1} = x_t + v · Δt · cos(heading)`
- 融合 IMU 加速度、陀螺仪角速度

**优点**：
- 不依赖 GPS
- 短时精度高

**缺点**：
- 误差累积（需 GPS 校正）
- 需 IMU 硬件支持
- 长断档失效

### 6.4 速度约束插值

**原理**：基于断档前后速度，估算合理位移。

**方法**：
- 取断档前速度 v1、断档后速度 v2
- 估算断档段平均速度 v = (v1 + v2) / 2
- 估算位移 d = v · Δt
- 在 P1、P2 连线上按 d 分配

**优点**：无需路网，考虑运动学

**缺点**：仍是直线，不考虑道路

### 6.5 大厂补偿方案

- **百度鹰眼**：提供 5 种补偿方式（直线插值、骑行路径、驾车路径、步行路径、爬升曲线）
- **腾讯轨迹云**：supplement 参数，直线与驾车两种
- **高德猎鹰**：attribute 补偿属性

### 6.6 补偿能力选型

本项目**不实现补偿**。原因：
1. 高质量补偿（路径规划插值）依赖绑路与路网
2. 线性插值质量低，价值有限
3. 航位推算需 IMU，超出 SDK 范围

## 9. 学术算法参考

### 9.1 经典论文

| 论文 | 年份 | 贡献 | 与本项目关系 |
|------|------|------|-------------|
| Douglas & Peucker, "Algorithms for the reduction of the number of points required to represent a digitized line or its caricature" | 1973 | DP 抽稀算法 | 抽稀算法参考 |
| Visvalingam & Whyatt, "Line generalisation by repeated elimination of points" | 1993 | VW 抽稀算法 | 抽稀算法参考 |
| Newson & Krumm, "Hidden Markov Map Matching Through Noise and Sparseness" | 2009 | HMM 绑路事实标准 | 绑路参考 |
| Kalman, "A New Approach to Linear Filtering and Prediction Problems" | 1960 | 卡尔曼滤波 | 去噪平滑参考 |
| Tukey, "Exploratory Data Analysis" | 1977 | 箱线图与 IQR | 异常检测参考 |

### 9.2 异常检测相关

- **3σ 准则**：正态分布下 99.7% 数据在 3σ 内，超出视为异常
- **修正 Z-score**：用 MAD（Median Absolute Deviation）替代 σ，对极端值稳健
- **Grubbs 检验**：假设检验框架下的异常检测
- **ESD（Generalized Extreme Studentized Deviate）**：适用于未知异常个数场景

本项目采用经典 Z-score 与 IQR，兼顾实现简单与效果。

### 9.3 轨迹平滑相关

- **Savitzky-Golay 滤波**：多项式拟合平滑，保留信号特征
- **样条插值**：平滑曲线拟合
- **小波去噪**：频域分析

这些方法改变原始点位置，与本项目"保留有效原始点"目标不符，故不采用。

### 9.4 伪静止识别相关

- **Stay Point Detection**（Li et al., 2008）：基于时间与距离阈值识别停留点，经典算法
- 本项目伪静止状态机借鉴 Stay Point Detection 思想，但强调状态机平滑（输出连续状态），而非仅识别停留点。

### 9.5 算法理论小结

本项目的算法选型遵循"够用、稳健、可解释、低复杂度"原则：
1. 不追求学术前沿，选用成熟经典方法
2. 优先保证可解释性（运营可调参）
3. 控制实现复杂度（双语言对等）

## 10. 本项目简化方案的选型理由

### 10.1 能力范围取舍

| 能力 | 是否实现 | 选型理由 |
|------|---------|---------|
| 精度过滤 | 是 | 纯数值判断，零依赖，效果直接 |
| 伪静止状态机 | 是 | 解决静止抖动这一高频痛点，状态机平滑优于单点判定 |
| Z-score 异常检测 | 是 | 实现简单，统计意义明确，冷启动 + 滑动窗口适应轨迹变化 |
| IQR 异常检测 | 是 | 作为 Z-score 备选，对偏态分布稳健 |
| 卡尔曼滤波 | 否 | 改变原始点位置，调参难，双语言对等复杂 |
| Douglas-Peucker 抽稀 | 否（预留） | 当前需求聚焦去噪，DP 作为扩展 Stage 预留 |
| 绑路 | 否 | 强依赖路网，属大厂能力，开源方案已成熟 |
| 补偿 | 否 | 高质量补偿依赖绑路，低质量补偿价值有限 |

### 10.2 算法选型理由

#### 10.2.1 为什么用 Z-score 而非卡尔曼

1. **目标差异**：本项目目标是"清洗"（识别并处理飞点），保留有效原始点不变；卡尔曼是"平滑"（改变所有点位置）
2. **可解释性**：Z-score 阈值有明确统计含义，运营易调参；卡尔曼 Q、R 矩阵调参需专业背景
3. **实现复杂度**：Z-score 增量更新 O(1)；卡尔曼需维护状态矩阵
4. **双语言对等**：Z-score 数值计算简单，易保证 Go / Java 一致；卡尔曼矩阵运算易引入库差异

#### 10.2.2 为什么保留 IQR 作为备选

1. Z-score 假设正态分布，实际轨迹速度分布常偏态
2. IQR 基于分位数，对偏态稳健
3. 运营可根据数据分布选择
4. 实现成本可控

#### 10.2.3 为什么用状态机而非 Stay Point Detection

1. Stay Point Detection 是"识别"停留点，输出停留点列表
2. 本项目需求是"清洗"轨迹，输出连续清洗后的点序列
3. 状态机输出连续状态（MOVING / PENDING_STATIC / STATIC），可直接驱动点级处理（静止段仅留锚点）
4. 状态机支持流式增量处理（Stay Point Detection 通常需完整轨迹）

#### 10.2.4 为什么不做绑路与补偿

1. **数据依赖**：绑路需路网数据（GB 级 OSM 数据），与"轻量级 SDK"定位冲突
2. **工程量**：HMM + Viterbi + 路网索引 + 最短路径，工程量是去噪的 10 倍以上
3. **大厂壁垒**：绑路精度依赖路网质量与更新频率，大厂有数据优势，自建难超越
4. **开源成熟**：OSRM / GraphHopper / Valhalla 已提供工业级绑路，无需重复造轮
5. **职责单一**：SDK 聚焦去噪 + 抽稀，与开源绑路引擎互补，职责清晰

### 10.3 与大厂方案的差异定位

| 维度 | 大厂方案 | 本项目 |
|------|---------|--------|
| 部署形态 | 云服务 | 嵌入式 SDK |
| 数据流转 | 上传云端 | 本地处理 |
| 能力范围 | 全链路（去噪 + 抽稀 + 绑路 + 补偿） | 去噪 + 抽稀 |
| 路网依赖 | 是 | 否 |
| 离线可用 | 否 | 是 |
| 定制化 | 受限 | 全参数可调 |
| 成本 | 按量计费 | 免费 |
| 适用场景 | 大规模、需绑路 | 离线、敏感数据、边缘端 |

### 10.4 与开源方案的协作建议

推荐架构：

```
原始 GPS 数据
     │
     ▼
┌──────────────────────┐
│ 本项目（去噪 + 抽稀） │  离线、本地、零依赖
└──────────────────────┘
     │ 清洗后轨迹
     ▼
┌──────────────────────┐
│ 开源绑路引擎          │  OSRM / GraphHopper
│ （可选，需路网）      │
└──────────────────────┘
     │ 绑路后轨迹
     ▼
下游业务（里程 / 回放 / 围栏）
```

清洗在前、绑路在后的顺序原因：
1. 清洗先去除飞点，避免飞点污染绑路的 HMM
2. 抽稀减少点数，降低绑路计算量
3. 清洗后轨迹更平滑，绑路候选路段更准

## 11. 调研结论

### 11.1 核心结论

1. **轨迹清洗四大能力**（去噪、抽稀、绑路、补偿）中，去噪与抽稀可纯算法实现，零外部依赖；绑路与补偿强依赖路网数据。
2. **大厂方案**以云端全链路服务为主，绑路是核心壁垒；本项目不与大厂在绑路上竞争，而是在去噪 + 抽稀上提供离线、可嵌入、可定制的补充。
3. **异常检测算法**推荐 Z-score（主）+ IQR（备），兼顾实现简单、统计稳健、可解释；卡尔曼滤波因改变原始点位置不适合清洗场景。
4. **伪静止处理**采用状态机优于单点判定，支持流式且输出连续状态。
5. **绑路与补偿**建议上层接入开源引擎（OSRM / GraphHopper / Valhalla），本项目不实现。

### 11.2 对需求与设计的输入

本调研结论直接驱动 [01-requirements.md](./01-requirements.md) 的能力范围界定与 [03-design.md](./03-design.md) 的算法选型：
- 需求文档的约束条件（仅去噪 + 抽稀）源于第 10 节取舍
- 设计文档的 Stage 3 异常检测双方法（Z-score + IQR）源于第 3 节对比
- 设计文档的伪静止状态机源于第 9.4 节与 10.2.3 节

### 11.3 后续工作

1. **算法验证**：在公开数据集（Seattle、GeoLife）上验证 Z-score / IQR 的 Precision / Recall，详见 [04-validation.md](./04-validation.md)
2. **参数调优**：网格搜索关键参数（ZThreshold、MinStaticPoints 等），给出推荐配置
3. **扩展预留**：为 DP 抽稀、绑路接入预留 Stage 接口
